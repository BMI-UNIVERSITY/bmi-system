# Contributing to BMI University

Thank you for your interest in contributing to the BMI University web application!

## 🛠️ Development Workflow

1. **Fork or Clone**: Ensure you have access to the repository.
2. **Branch**: Create a new branch for your feature or bug fix (`git checkout -b feature/your-feature-name`).
3. **Commit**: Make sure your commits are descriptive.
4. **Push**: Push your changes to your branch.
5. **Pull Request**: Open a pull request against the `master` or `main` branch. Use the provided Pull Request template to describe your changes.

## 🤝 Contract Changes

Any PR that modifies `app/apply/page.jsx` (especially how it redirects to `apply.bmiuniversity.org`) **must** update `CHANGELOG-CONTRACT.md` and open a linked issue in the `bmi-portal` repository. The portal backend team must review any change touching the integration surface.

## 🐞 Reporting Bugs

If you find a bug, please create an issue using the Bug Report template. Include as much detail as possible to help us reproduce and fix the issue.

## 💡 Proposing Features

We love new ideas! If you have a feature request, please open an issue and tag it as an enhancement. Describe the use case and how it benefits BMI University.
